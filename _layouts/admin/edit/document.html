Help

🏁 Getting started

All documents that have been uploaded to the document store repository are listed on this page.

The documents are arranged by their parent folder. Selecting the name of the folder will reveal a list of all the documents in that folder.

Any files not placed into folders will display under “unfiled”.

If you have recently uploaded a new document to the repository, it can take several minutes to appear in this list.

🔗 Getting the link to a document

This tool will allow you to get link to files stored in the documents repository.

Find the file you want to link to and use the “Copy file URL” button to copy the link to your clipboard. Paste the link using Ctrl + V (Windows) or Cmd + V (MacOS).

Links should start with www.civilservice.lgbt.

📁 Upload files to a new folder

You can upload a new document to an existing folder by selecting the folder you want to use and using the ”Upload here” button. This will take you to the correct page to the correct page on Github.

Once you have uploaded the file on GitHub, it will take several minutes to appear in this list. You can then link to it once it's listed on this page.

📂 Upload files to an existing folder

You can upload a new document to an existing folder by selecting the folder you want to use and using the ”Upload here” button. This will take you to the correct page to the correct page on Github.

Once you have uploaded the file on GitHub, it will take several minutes to appear in this list. You can then link to it once it's listed on this page.

Important: All files must be placed inside the top level “/store” folder. If they are not, they will not show up in this list and may fail to be publicly accessible.

🔐 GitHub permissions

You will require a GitHub account and relevant permissions to the document store repository in order to add or amend its content.
